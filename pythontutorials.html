Functions

Functions are important to learn first so one can practice challenges that return values. That way, tests can be easy through assertion.

Functions - Basic Syntax

Functions are defined with the def keyword followed by the name of the function with parenthesis and a semi colon.
def foo():
def foo_bar():
The body of the function comes after the semi colon, a \n (newline) and a tab. 

def foo():
    print("Hello World!")

Functions - Arguments and Inputs to the Function

def print_argument(argument):
    print(argument))

Arguments are objects that are passed into a function, like print_argument(argument) above.

Python is dynamically typed, meaning the one writing the function does not need to supply the argument's type like in static languages. There is, however, the option to *add types by using a colon after the argument and the type. 

# Argument arg using type string.
def foo( arg: str ):     
    print("Hello World!")

Functions - Multiple Positional Arguments

def print_argument(*args):
    print( argument )

Multiple arguments can be used even when one does not know how many the function will receive.

One can use the * operator before the parameter and that tells python to accept as many arguments that are supplied by position and not keyword. It then creates a tuple with the inputs and thats how they are accessed. 

def foo( *args ):
    for i in args:
        print(i)

Then, one can call foo() with as many inputs as needed.

foo(1,2,3,4) 

1
2
3
4

Functions - Multiple Keyword Arguments
def foo(*args, **kwargs):
    for k,v in kwargs.items():
        print(k, v)

foo(1, 2, 3, one=1, two=2)
# one 1
# two 2

There is a lot going on above. The main thing to focus on is **kwargs.

This stands for keyword arguments and is stored in a dictionary called kwargs. This is arbitrary meaning one picks the name; it can be kwargs or whatever one wants.

Accesing the keyword arguments is as simple as going through the kwargs.items() or accesing it by value.
