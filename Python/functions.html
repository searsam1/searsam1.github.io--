Functions

Functions are important to learn first so one can practice challenges that return values. That way, tests can be easy through assertion.

Functions in python are similar to mathematical functions: take in a value, modify it and output the result. Though, when it comes to coding and programming, functions are more a way to copy and paste code with ease. One is taking a set of instructions and putting it inside a function then executing those steps repeatedly.

It is like when cooking and using pre-made cookie mix. Throw in some water to the batch and they are ready to be cooked. The point is, one does not have to mix together flour, water, eggs etc. to make them, it is already mixed. Functions work in a similar way. Instead of rewriting the function every time, it is called with ().

Basic Syntax

Functions are defined with the def keyword followed by the name of the function with parenthesis and a semi colon.
def foo():
def foo_bar():
The body of the function comes after the semi colon, a \n (newline) and a tab. 

def foo():
    print("Hello World!")

Defining the function will not call the function. The function must be executed with parenthesis: foo() .

Arguments and Inputs to the Function

def print_argument(argument):
    print(argument))

Arguments are objects that are passed into a function, like print_argument(argument) above.

Python is dynamically typed, meaning the one writing the function does not need to supply the argument's type like in static languages. There is, however, the option to *add types by using a colon after the argument and the type. 

# Argument arg using type string.
def foo( arg: str ):     
    print("Hello World!")

Multiple Positional Arguments

def print_argument(*args):
    print( argument )

Multiple arguments can be used even when one does not know how many the function will receive.

One can use the * operator before the parameter and that tells python to accept as many arguments that are supplied by position and not keyword. It then creates a tuple with the inputs and thats how they are accessed. 

def foo( *args ):
    for i in args:
        print(i)

Then, one can call foo() with as many inputs as needed.

foo(1,2,3,4) 

1
2
3
4

Multiple Keyword Arguments
def foo(*args, **kwargs):
    for k,v in kwargs.items():
        print(k, v)

foo(1, 2, 3, one=1, two=2)
# one 1
# two 2

There is a lot going on above. The main thing to focus on is **kwargs.

This stands for keyword arguments and is stored in a dictionary called kwargs. This is arbitrary meaning one picks the name; it can be kwargs or whatever one wants.

Accessing the keyword arguments is as simple as going through the kwargs.items() or accessing it by value.

Default Arguments

def foo(arg="default"):
    print(arg)

Setting an argument equal to an initial value is called giving it a default argument. This makes the argument optional.

Calling foo() without any arguments will print "default", while calling foo with an argument like foo("happy"), it will print the string "happy".

Returning Values

Returning values makes functions in python a bit more mathematical. Say I have a simple function,
f(x) = x * x.

f(3) = 3 * 3 = 9

It math it is implied that the function is returned. f(3) = 9 and one did not have to write return x at the end.

With programming it is different: one must specify the return value of the function, or None will be returned. So far, the functions used have all printed something rather than returning it. Once a value is returned the function is killed. 

def f(x):
    return x * x         
    
print( f(3) )    
# -> 9

f(3) returns 9, which is then printed out.

It helps to understand when assigning the return result to a variable. result = f(3). result will take on the value of 9 and can even be put into another function to be manipulated.
